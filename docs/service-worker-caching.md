# Service Worker 캐싱 정책 (이 블로그)

`lib/sw.ts` · `public/sw.js` · `components/layout/service-worker-register.tsx` 기준으로 정리한 메모입니다.

---

## 정책 체크리스트

- [x] **문서 네비게이션**(브라우저가 HTML 문서를 받는 최상위 이동, `navigate` / `destination === "document"`)은 **SWR을 쓰지 않고**, 다른 GET과 같이 **네트워크 우선**입니다. 응답이 `ok`이면 문서와 HTML에서 뽑은 **`/_next/static` 묶음**을 캐시에 넣고, 네트워크가 실패하면 **캐시된 HTML이 가리키는 정적 자산이 모두 있을 때만** 캐시된 문서를 돌려줍니다. (아니면 `offline.html`.)
- [x] **`/_next/static`** 과 **RSC(`?_rsc=` 등)** 는 **네트워크 우선**, 성공 시 캐시에 넣고, **실패(오프라인 등) 시에만 캐시**를 씁니다.
- [x] **(선택) 업데이트 알림**: 새 SW 가 대기 상태가 되면 앱 쪽에서 **토스트로 새로고침 유도** 등을 할 수 있습니다. (`SKIP_WAITING` 메시지 등)

---

## 알려진 이슈: 오프라인에서 HTML만 보이고 CSS가 안 입혀지는 경우

**증상**  
오프라인(또는 네트워크 실패 직후)에 문서(HTML)는 캐시에서 나오는데, `/_next/static/...css` 요청은 캐시에 없어 **스타일시트가 비어** 화면이 “무스타일 HTML”처럼 보일 수 있습니다.

**원인**  
문서는 SWR로 **오래된 HTML이라도 즉시** 줄 수 있지만, 그 HTML이 가리키는 CSS·JS는 별도 요청입니다. 이 자산들이 **한 번도 온라인에서 캐시에 들어가지 않았다면** 오프라인에서는 네트워크도 막혀 있어 **문서와 정적 자산의 준비 시점이 어긋납니다.**

**이 저장소에서의 완화**

1. **문서와 `/_next/static` 묶음 캐시** (`lib/sw.ts`)  
   문서는 **항상 네트워크를 먼저** 시도하고, 응답이 성공하면 캐시에 넣을 때 본문에서 **`/_next/static/`** 을 가리키는 `link`·`script` URL을 정규식으로 뽑아 **`cache.add`로 같이 넣습니다.** (최대 80개) “한 번 온라인에서 받은 페이지”의 HTML과 초기 번들이 한 세트로 쌓이게 합니다.

2. **오프라인 문서 가드** (`lib/sw.ts`)  
   오프라인에서 캐시된 HTML을 돌려주기 **전에**, 그 HTML이 가리키는 **`/_next/static` 참조가 캐시에 전부 있는지** 검사합니다. 하나라도 없으면 캐시 HTML 대신 **`offline.html`(인라인 스타일)** 을 돌려 무스타일 노출을 피합니다. (초기 HTML에 `script`/`link`로 안 잡히는 **지연 로드 청크**는 이 목록에 없을 수 있어, 오프라인에서 일부 동작이 비는 여지는 남습니다.)

3. **온라인 시 보조 워밍** (`components/layout/service-worker-register.tsx`)  
   DOM에 붙은 `/_next/static`의 `.css`/`.js` URL을 `CACHE_URLS`로 SW에 넘겨 **추가로** 소량 캐시합니다. (1번과 중복될 수 있어 보조에 가깝습니다.)

**한계**  
HTML 정규식은 **첫 응답에 등장하는** `link`/`script` 위주입니다. 런타임에만 로드되는 청크까지 전부 맞추려면 빌드 산출물 목록 기반 프리캐시 등 **정책·용량 트레이드오프**가 또 필요합니다.

## 관련 파일

| 파일                                            | 역할                                                                                                 |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `lib/sw.ts`                                     | SW 소스. 문서 SWR, 정적 자산 네트워크 우선·캐시, 오프라인 가드, `CACHE_URLS` / `SKIP_WAITING` 메시지 |
| `scripts/generate-sw-precache.ts`               | 모티프·오프라인 HTML 생성물 등 (빌드 시 `pnpm build-sw`)                                             |
| `public/sw.js`                                  | esbuild 번들 결과 (배포물)                                                                           |
| `components/layout/service-worker-register.tsx` | 등록, 오프라인 토스트, CSS 워밍 메시지 등 클라이언트 보조                                            |
